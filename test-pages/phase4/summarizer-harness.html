HoneyLLM #47 — Summarizer API vs Prompt API evaluation harness

Side-by-side comparison of the current generic Prompt API path used by src/probes/summarization.ts versus Chrome's purpose-built, LoRA-fine-tuned Summarizer API (Summarizer.create({type, format, length})). For each of the 9 Phase 2 / Phase 3 fixtures, both paths receive the same input text. The harness records raw output, latency, and four heuristic compliance signals (compromise phrase, leaked-prompt phrase, exfil URL, AI self-reference / role adoption).

Read-only evaluation - does not modify the production probe. Requires Chrome with both APIs available (window.LanguageModel + window.Summarizer or self.Summarizer).
